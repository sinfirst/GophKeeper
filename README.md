# GophKeeper – Менеджер паролей и приватных данных

GophKeeper — это клиент-серверное приложение для безопасного хранения и синхронизации конфиденциальной информации: логинов/паролей, текстовых заметок, бинарных файлов и данных банковских карт. Проект реализован на языке Go с использованием gRPC для взаимодействия между клиентом и сервером.

## Возможности

- Регистрация, аутентификация и авторизация пользователей.
- Хранение данных различных типов:
  - Пары логин/пароль;
  - Произвольные текстовые данные;
  - Бинарные данные (файлы);
  - Данные банковских карт.
- Добавление произвольной метаинформации к любым данным.
- Синхронизация данных между несколькими клиентами одного пользователя.
- CLI-клиент с поддержкой Windows, Linux и macOS.
- TUI (терминальный интерфейс) — опционально.
- Поддержка OTP (одноразовые пароли) — опционально.
- Использование бинарного протокола (gRPC).
- Юнит-тесты с покрытием не менее 80%.

## Архитектура

Проект состоит из двух основных частей: сервер и клиент. Взаимодействие осуществляется через защищённое соединение (TLS) по протоколу gRPC.

```
.
├── cmd
│   ├── client          # точка входа для клиента
│   └── server          # точка входа для сервера
├── internal
│   ├── app             # бизнес-логика приложения
│   ├── client          # логика клиента (работа с API сервера, локальное хранение)
│   ├── config          # конфигурация (переменные окружения, флаги)
│   ├── handlers        # gRPC-обработчики (серверная сторона)
│   ├── middleware      # промежуточное ПО (аутентификация, логирование)
│   ├── models          # модели данных, общие для клиента и сервера
│   ├── storage         # работа с хранилищем (БД, миграции)
│   └── tui              # терминальный интерфейс (опционально)
├── proto
│   └── gophkeeper      # Protobuf-спецификации и сгенерированный код gRPC
├── go.mod
├── go.sum
└── README.md
```

### Основные компоненты

- **Сервер** (`cmd/server`): обрабатывает запросы клиентов, управляет пользователями и данными, использует СУБД (PostgreSQL).
- **Клиент** (`cmd/client`): CLI/TUI приложение, выполняет локальное шифрование данных, синхронизируется с сервером, сохраняет данные в локальном зашифрованном хранилище.
- **Протокол**: gRPC + Protobuf (папка `proto/gophkeeper`). Аутентификация через JWT.
- **Хранилище**: на сервере — реляционная база данных (миграции в `internal/storage/migrations`). На клиенте — зашифрованное локальное хранилище .
- **Безопасность**: передача по TLS, шифрование данных на клиенте перед отправкой, хеширование паролей (bcrypt).

## Технологии

- Язык: Go 1.21+
- Протокол: gRPC, Protobuf
- База данных: PostgreSQL (или SQLite для локального хранения)
- Аутентификация: JWT
- TUI: ручной
- Тестирование: встроенный пакет `testify`, mock-сервер gRPC

## Установка и запуск

### Предварительные требования

- Установленный Go (версия 1.21 или новее)
- Установленная СУБД (PostgreSQL) для сервера

### Сборка

Клонируйте репозиторий:

```bash
git clone https://github.com/yourusername/gophkeeper.git
cd gophkeeper
```

Соберите сервер и клиент:

```bash
# сервер
go build -o bin/server cmd/server/main.go

# клиент
go build -o bin/client cmd/client/main.go
```

### Настройка

#### Сервер

1. Создайте базу данных.
2. Укажите параметры подключения к БД, порт сервера и другие настройки через переменные окружения или файл `.env`.

Пример `.env` для сервера:

```env
SERVER_PORT=8080
DATABASE_DSN=postgres://user:pass@localhost/gophkeeper?sslmode=disable
JWT_SECRET=supersecret
TLS_CERT=path/to/cert.pem
TLS_KEY=path/to/key.pem
```

Запустите сервер:

```bash
./bin/server
```

#### Клиент

Клиент может работать в двух режимах: CLI и TUI. Настройки клиента также задаются через переменные окружения или конфиг-файл.

```env
SERVER_ADDR=localhost:8080
TLS_CERT=path/to/server/ca.pem   # для проверки сертификата сервера
```

Запуск клиента:

```bash
# CLI режим
./bin/client --help

# TUI режим (если поддерживается)
./bin/client tui
```

## Использование

### Регистрация нового пользователя

```bash
./bin/client register --login alice --password secret
```

### Вход в систему

```bash
./bin/client login --login alice --password secret
```

После успешного входа токен аутентификации сохраняется локально.

### Управление данными

Примеры команд (зависят от реализации):

- Добавить пару логин/пароль:
  ```bash
  ./bin/client add login --resource example.com --login user --password pass --meta "personal"
  ```
- Показать все записи:
  ```bash
  ./bin/client list
  ```
- Получить конкретную запись:
  ```bash
  ./bin/client get <id>
  ```
- Синхронизация с сервером:
  ```bash
  ./bin/client sync
  ```

### TUI (если реализовано)

При запуске `./bin/client tui` открывается интерактивный интерфейс с панелями для навигации по данным, добавления/редактирования/удаления.

## Конфигурация

Основные параметры конфигурации:

| Переменная       | Описание                                    | По умолчанию       |
|------------------|---------------------------------------------|--------------------|
| `SERVER_ADDR`    | Адрес gRPC сервера (хост:порт)              | `localhost:8080`   |
| `DATABASE_DSN`   | Строка подключения к БД (только для сервера)| —                  |
| `JWT_SECRET`     | Секрет для подписи JWT                       | —                  |
| `TLS_CERT`       | Путь к сертификату TLS (сервер/клиент)       | —                  |
| `TLS_KEY`        | Путь к ключу TLS (сервер)                    | —                  |
| `LOG_LEVEL`      | Уровень логирования (debug, info, warn, error)| `info`            |

## Тестирование

Запуск юнит-тестов:

```bash
go test ./... -cover
```

Для просмотра покрытия в HTML:

```bash
go test ./... -coverprofile=coverage.out
go tool cover -html=coverage.out
```
